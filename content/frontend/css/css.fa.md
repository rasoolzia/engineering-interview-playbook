## 🧠 سوال ۱

**آیدی**: css-001  
**عنوان**: مدل جعبه ای CSS چیست و اجزای آن کدامند؟  
**سطح دشواری**: آسان  
**دسته‌بندی**: اصول اولیه

### پاسخ 📄

مدل جعبه ای CSS نحوه ساختار و فاصله گذاری عناصر در چیدمان را توصیف میکند.

هر عنصر به صورت یک جعبه مستطیلی نمایش داده میشود که شامل:

1. **Content** – محتوای واقعی متن یا رسانه
2. **Padding** – فاصله بین محتوا و حاشیه
3. **Border** – دور پدینگ و محتوا را احاطه میکند
4. **Margin** – فاصله خارج از حاشیه

به طور پیشفرض، `width` و `height` فقط روی جعبه محتوا اعمال میشوند.

با استفاده از:

```css
box-sizing: border-box;
```

محاسبه تغییر میکند تا پدینگ و بوردر در عرض و ارتفاع اعلام شده لحاظ شوند.

درک مدل جعبه ای برای کنترل چیدمان ضروری است.

## 🧠 سوال ۲

**آیدی**: css-002  
**عنوان**: تفاوت بین مقادیر `block`، `inline` و `inline-block` در display چیست؟  
**سطح دشواری**: آسان  
**دسته‌بندی**: چیدمان

### پاسخ 📄

### `block`

- کل عرض موجود را اشغال میکند
- از خط جدید شروع میشود
- میتوان عرض و ارتفاع تنظیم کرد

مثالها: `div`، `p`، `section`

### `inline`

- فقط به اندازه محتوا عرض میگیرد
- از خط جدید شروع نمیشود
- نمیتوان عرض و ارتفاع تنظیم کرد

مثالها: `span`، `a`، `strong`

### `inline-block`

- از خط جدید شروع نمیشود
- میتوان عرض و ارتفاع تنظیم کرد
- پدینگ و مارجین را رعایت میکند

زمانی استفاده میشود که عناصر نیاز به موقعیت inline اما رفتار block دارند.

## 🧠 سوال ۳

**آیدی**: css-003  
**عنوان**: اختصاصیت (Specificity) در CSS چیست و چگونه محاسبه میشود؟  
**سطح دشواری**: متوسط  
**دسته‌بندی**: اصول اولیه

### پاسخ 📄

اختصاصیت تعیین میکند کدام قانون CSS زمانی که چندین قانون یک عنصر را هدف میگیرند، اعمال شود.

سلسله مراتب اختصاصیت:

1. استایل های Inline
2. IDها
3. کلاسها، ویژگیها، شبهکلاسها
4. عناصر و شبهعناصر

مثال:

```css
#main {
  color: red;
}
.container p {
  color: blue;
}
```

سلکتور ID برنده میشود چون اختصاصیت بالاتری دارد.
اختصاصیت به صورت یک مقدار چهار بخشی محاسبه میشود:
Inline → IDs → Classes → Elements
مقدار بالاتر برنده است. اگر برابر باشند، قانون بعدی در استایلشیت اعمال میشود.

## 🧠 سوال ۴

**آیدی**: css-004  
**عنوان**: تفاوت بین `position: relative`، `absolute`، `fixed` و `sticky` چیست؟  
**سطح دشواری**: متوسط  
**دسته‌بندی**: چیدمان

### پاسخ 📄

### `relative`

- نسبت به موقعیت عادی خود قرار میگیرد
- در جریان سند باقی میماند

### `absolute`

- از جریان سند خارج میشود
- نسبت به نزدیکترین والد موقعیتدار قرار میگیرد

### `fixed`

- نسبت به ویوپورت قرار میگیرد
- در حین اسکرول حرکت نمیکند

### `sticky`

- مانند `relative` رفتار میکند تا رسیدن به آستانه
- سپس مانند `fixed` در داخل کانتینر خود رفتار میکند

موقعیتدهی، لایه بندی و رفتار چیدمان عناصر را کنترل میکند.

## 🧠 سوال ۵

**آیدی**: css-005  
**عنوان**: تفاوت بین Flexbox و Grid چیست؟  
**سطح دشواری**: متوسط  
**دسته‌بندی**: چیدمان

### پاسخ 📄

Flexbox و Grid سیستم های چیدمان هستند اما مشکلات متفاوتی را حل میکنند.

### Flexbox

- چیدمان یکبعدی
- چیدمان را در یک ردیف یا ستون کنترل میکند
- بهترین گزینه برای تراز و توزیع آیتمها

استفاده برای:

- نوار ناوبری
- مرکز کردن محتوا
- چیدمان کامپوننت های کوچک

### Grid

- چیدمان دوبعدی
- ردیفها و ستونها را به طور همزمان کنترل میکند
- برای چیدمان کامل صفحه طراحی شده

استفاده برای:

- ساختارهای پیچیده صفحه
- گریدهای واکنشگرا
- سیستم های چیدمان

**Flexbox محتوا-محور است.**  
**Grid چیدمان-محور است.**

## 🧠 سوال ۶

**آیدی**: css-006  
**عنوان**: آبشار CSS چیست و مرورگر چگونه تصمیم میگیرد کدام قانون را اعمال کند؟  
**سطح دشواری**: متوسط  
**دسته‌بندی**: اصول اولیه

### پاسخ 📄

آبشار CSS الگوریتمی است که برای تعیین اینکه کدام قانون استایل زمانی که چندین قانون یک عنصر را هدف میگیرند، اعمال شود، استفاده میشود.

مرورگر تعارضات را به این ترتیب حل میکند:

1. **منشأ** (user agent، استایل های کاربر، استایل های نویسنده)
2. **اهمیت** (`!important`)
3. **اختصاصیت**
4. **ترتیب منبع** (آخرین اعلام شده برنده است)

درک آبشار برای رفع اشکال رفتار غیرمنتظره استایل دهی ضروری است.

## 🧠 سوال ۷

**آیدی**: css-007  
**عنوان**: زمینه انباشته (stacking context) چیست و چگونه ایجاد میشود؟  
**سطح دشواری**: سخت  
**دسته‌بندی**: چیدمان

### پاسخ 📄

زمینه انباشته نحوه لایه بندی عناصر در امتداد محور z را کنترل میکند.

یک زمینه انباشته جدید زمانی ایجاد میشود که:

- عنصر `position` غیر از static داشته باشد و `z-index` تنظیم شده باشد
- `opacity` کمتر از 1 باشد
- `transform` اعمال شده باشد
- از `filter`، `perspective`، `will-change` و غیره استفاده شود

عناصر داخل یک زمینه انباشته مستقل از عناصر خارج از آن لایه بندی میشوند.

درک زمینه انباشته برای حل مشکلات z-index بسیار مهم است.

## 🧠 سوال ۸

**آیدی**: css-008  
**عنوان**: تفاوت بین `overflow: hidden`، `auto` و `scroll` چیست؟  
**سطح دشواری**: متوسط  
**دسته‌بندی**: چیدمان

### پاسخ 📄

ویژگی `overflow` کنترل میکند که چه اتفاقی میافتد وقتی محتوا از کانتینر خود فراتر میرود.

- `hidden` → محتوا برش میخورد، بدون نوار اسکرول
- `auto` → نوار اسکرول فقط در صورت نیاز ظاهر میشود
- `scroll` → نوار اسکرول همیشه قابل مشاهده است

overflow همچنین میتواند یک زمینه قالببندی بلوکی جدید (BFC) ایجاد کند که بر رفتار چیدمان تأثیر میگذارد.

## 🧠 سوال ۹

**آیدی**: css-009  
**عنوان**: زمینه قالببندی بلوکی (BFC) چیست و چرا مفید است؟  
**سطح دشواری**: سخت  
**دسته‌بندی**: چیدمان

### پاسخ 📄

زمینه قالببندی بلوکی (BFC) مکانیزم چیدمانی است که نحوه تعامل عناصر بلوکی را کنترل میکند.

یک BFC زمانی ایجاد میشود که:

- `overflow` مقدار visible نداشته باشد
- `display: flow-root`
- از `float` استفاده شود
- `position: absolute` یا `fixed`

مزایا:

- از فروپاشی مارجین جلوگیری میکند
- عناصر شناور را محدود میکند
- محیطهای چیدمان ایزوله ایجاد میکند

BFC اغلب برای رفع باگهای چیدمان مربوط به float ها یا مارجینهای فروپاشیده استفاده میشود.

## 🧠 سوال ۱۰

**آیدی**: css-010  
**عنوان**: فروپاشی مارجین (margin collapsing) چیست و چه زمانی رخ میدهد؟  
**سطح دشواری**: متوسط  
**دسته‌بندی**: چیدمان

### پاسخ 📄

فروپاشی مارجین زمانی رخ میدهد که مارجینهای عمودی عناصر بلوکی مجاور در یک مارجین واحد ترکیب شوند.

زمانی اتفاق میافتد که:

- دو عنصر بلوکی خواهر و برادر مارجین عمودی داشته باشند
- یک والد و اولین/آخرین فرزندش هر دو مارجین عمودی داشته باشند

مارجین نتیجه برابر با بزرگترین مقدار است، نه مجموع آنها.

فروپاشی مارجین بین خواهر و برادرهای مجاور در کانتینرهای flex یا grid رخ نمیدهد (چون آنها زمینه قالب بندی جدید ایجاد میکنند).

## 🧠 سوال ۱۱

**آیدی**: css-011  
**عنوان**: تفاوت بین واحدهای `em`، `rem`، `%`، `vh` و `vw` چیست؟  
**سطح دشواری**: متوسط  
**دسته‌بندی**: اصول اولیه

### پاسخ 📄

CSS واحدهای نسبی متعددی ارائه میدهد.

- `em` → نسبت به font-size والد
- `rem` → نسبت به font-size ریشه (`html`)
- `%` → نسبت به عنصر والد
- `vh` → 1% از ارتفاع ویوپورت
- `vw` → 1% از عرض ویوپورت

از `rem` برای مقیاسبندی یکنواخت در کامپوننت ها استفاده کنید.  
از واحدهای ویوپورت برای چیدمانهای واکنشگرا استفاده کنید.

## 🧠 سوال ۱۲

**آیدی**: css-012  
**عنوان**: تفاوت بین `min-width`، `max-width` و `width` چیست؟  
**سطح دشواری**: متوسط  
**دسته‌بندی**: چیدمان

### پاسخ 📄

- `width` عرض پایه یک عنصر را تنظیم میکند
- `min-width` حداقل عرض مجاز را تعریف میکند
- `max-width` حداکثر عرض مجاز را تعریف میکند

اگر محدودیتها با هم تعارض داشته باشند:

- `min-width` بر `max-width` اولویت دارد
- `min-width` میتواند `width` را نادیده بگیرد

این ویژگیها برای طراحی واکنشگرا ضروری هستند.

## 🧠 سوال ۱۳

**آیدی**: css-013  
**عنوان**: تفاوت بین `reflow` (چیدمان) و `repaint` در رندر CSS چیست؟  
**سطح دشواری**: سخت  
**دسته‌بندی**: عملکرد

### پاسخ 📄

وقتی استایلها تغییر میکنند، مرورگر ممکن است موارد زیر را فعال کند:

### Reflow (چیدمان)

- موقعیت و ابعاد عناصر را دوباره محاسبه میکند
- هزینهبرتر است
- توسط تغییرات در ویژگی های تأثیرگذار بر چیدمان فعال میشود (width، height، position)

### Repaint

- ظاهر بصری را بدون تغییر چیدمان بهروزرسانی میکند
- هزینه کمتری دارد
- توسط تغییراتی مانند رنگ یا پسزمینه فعال میشود

reflow های مکرر میتوانند به طور قابل توجهی به عملکرد آسیب برسانند.

بهینهسازی CSS شامل به حداقل رساندن تغییرات فعالکننده چیدمان است.

مرورگرهای مدرن repaint را تا حد زیادی بهینه میکنند، اما reflow بیش از حد (خصوصاً در حلقه ها) میتواند باعث layout thrashing شود.

## 🧠 سوال ۱۴

**آیدی**: css-014  
**عنوان**: ویژگی های سفارشی CSS (متغیرهای CSS) چیستند و مزایای آنها چیست؟  
**سطح دشواری**: متوسط  
**دسته‌بندی**: اصول اولیه

### پاسخ 📄

ویژگی های سفارشی CSS به شما اجازه میدهند مقادیر را در سراسر استایلشیت خود ذخیره و استفاده مجدد کنید.

سینتکس:

```css
:root {
  --primary-color: #3498db;
  --spacing: 16px;
}

.button {
  background: var(--primary-color);
  padding: var(--spacing);
}
```

مزایا:

- میتوانند به صورت پویا با JavaScript تغییر کنند
- از طریق آبشار به ارث میرسند
- به عناصر محدود میشوند (برخلاف متغیرهای پیشپردازنده)
- تممسازی و استایل دهی پویا را فعال میکنند

برخلاف متغیرهای Sass/Less، متغیرهای CSS در زمان اجرا ارزیابی میشوند.

## 🧠 سوال ۱۵

**آیدی**: css-015  
**عنوان**: ویژگی های منطقی CSS چیست و چرا مهم هستند؟  
**سطح دشواری**: متوسط  
**دسته‌بندی**: طراحی واکنشگرا

### پاسخ 📄

ویژگی های منطقی امکان کنترل چیدمان را بر اساس جهت نوشتار به جای جهت فیزیکی فراهم میکنند.

به جای:

```css
margin-left
padding-right
```

استفاده کنید از:

```css
margin-inline-start
padding-inline-end
```

مزایا:

- بین المللی سازی بهتر (پشتیبانی از RTL/LTR)
- چیدمان های سازگارتر
- طراحی واکنشگرای تمیزتر

ویژگی های منطقی چیدمان ها را جهت آگاه میکنند.

**آیدی**: css-016  
**عنوان**: لایه های آبشاری CSS (`@layer`) چیست و چرا معرفی شدند؟

### پاسخ 📄

مزایا:

لایه های آبشاری اولویتبندی ساختاریافته استایل را فراهم میکنند.

## 🧠 سوال ۱۷

**آیدی**: css-017  
**عنوان**: کوئریهای کانتینر چیست و چه تفاوتی با مدیا کوئری دارند؟  
**سطح دشواری**: سخت  
**دسته‌بندی**: طراحی واکنشگرا

### پاسخ 📄

مدیا کوئریها به اندازه ویوپورت واکنش نشان میدهند.

کوئریهای کانتینر به اندازه کانتینر والد واکنش نشان میدهند.

مثال:

```css
@container (min-width: 400px) {
  .card {
    flex-direction: row;
  }
}
```

> **_نکته:_** برای فعال شدن container query، والد باید `container-type: inline-size` (یا `size`) داشته باشد.

کوئریهای کانتینر طراحی واکنشگرای واقعاً مبتنی بر کامپوننت را ممکن میکنند.

آنها به کامپوننت ها اجازه میدهند بر اساس فضای موجود به جای اندازه صفحه سازگار شوند.

## 🧠 سوال ۱۸

**آیدی**: css-018  
**عنوان**: تفاوت بین `:nth-child()` و `:nth-of-type()` چیست؟  
**سطح دشواری**: متوسط  
**دسته‌بندی**: سلکتورها و اختصاصیت

### پاسخ 📄

`:nth-child(n)` یک عنصر را بر اساس موقعیتش در بین تمام خواهر و برادرها انتخاب میکند.

`:nth-of-type(n)` یک عنصر را بر اساس موقعیتش در بین خواهر و برادرهای همنوع انتخاب میکند.

مثال:

```css
p:nth-child(2)
p:nth-of-type(2)
```

اگر فرزند دوم یک `<p>` نباشد، `nth-child` با آن مطابقت نمییابد، اما `nth-of-type` ممکن است مطابقت یابد.

## 🧠 سوال ۱۹

**آیدی**: css-019  
**عنوان**: `will-change` چگونه بر عملکرد تأثیر میگذارد و چه زمانی باید استفاده شود؟  
**سطح دشواری**: سخت  
**دسته‌بندی**: عملکرد

### پاسخ 📄

ویژگی `will-change` مرورگر را از تغییرات آتی مطلع میکند.

مثال:

```css
will-change: transform;
```

به مرورگر اجازه میدهد رندر را بهینه کند (اغلب با ارتقا به لایه GPU).

اما:

- حافظه مصرف میکند
- استفاده بیش از حد میتواند عملکرد را کاهش دهد

به ندرت از آن استفاده کنید و پس از اتمام انیمیشن آن را حذف کنید.

**سطح دشواری**: سخت  
**دسته‌بندی**: طراحی واکنشگرا

### پاسخ 📄

واحد سنتی `vh` (viewport height) بر اساس large viewport size (lvh) است که میتواند در موبایل وقتی نوار آدرس نمایش/مخفی میشود، باعث پرش چیدمان شود.

- `svh` (small viewport height): 1% از کوچکترین ارتفاع ویوپورت (وقتی نوارهای UI کاملاً نمایان هستند)

- `dvh` (dynamic viewport height): به صورت پویا بین svh و lvh بهروزرسانی میشود وقتی عناصر UI تغییر میکنند (برای اکثر موارد توصیه میشود)

`vh` اکنون در مرورگرهای مدرن معادل `lvh` است.

از `dvh` برای چیدمان های تمام ارتفاع پایدار در موبایل (مثلاً بخش های hero) استفاده کنید تا از CLS جلوگیری شود.

**آیدی**: css-027  
**عنوان**: تفاوت بین `opacity: 0`، `visibility: hidden` و `display: none` را از نظر چیدمان، عملکرد، دسترس پذیری و تعامل توضیح دهید.  
**سطح دشواری**: سخت  
**دسته‌بندی**: اصول اولیه

### پاسخ 📄

این ویژگیها عناصر را مخفی میکنند اما رفتار متفاوتی دارند:

- `opacity: 0`: عنصر نامرئی است اما همچنان فضا اشغال میکند، در جریان باقی میماند، تعاملی است (قابل کلیک، قابل فوکوس)، و در transition/animation شرکت میکند. screen reader ها ممکن است آن را اعلام کنند.
- `visibility: hidden`: عنصر نامرئی است، فضا اشغال میکند، از tab order حذف میشود (غیرقابل فوکوس/تعامل)، اما همچنان در جریان DOM است. اگر opacity <1 تنظیم نشده باشد، stacking context ایجاد میکند.
- `display: none`: عنصر کاملاً از چیدمان حذف میشود (فضایی اشغال نمیکند)، رندر نمیشود، تعاملی نیست، در accessibility tree نیست (screen reader ها نادیده میگیرند). باعث reflow میشود.

از `display: none` برای محتوای شرطی استفاده کنید؛ `visibility: hidden` برای مخفی کردن با حفظ فضا؛ `opacity: 0` برای انیمیشن های fade.

### پاسخ 📄

`inline-flex` و `inline-grid` عنصر را inline-level میکنند (در inline formatting context شرکت میکند) اما یک flex/grid formatting context داخلی ایجاد میکنند.

رفتارهای کلیدی:

- مارجین های افقی (left/right) به طور عادی اعمال میشوند.

- مارجین های عمودی رعایت میشوند اما به همان شکل مارجین های block روی line height یا جریان inline اطراف تأثیر نمیگذارند.

این به این دلیل است که جعبه های inline-level به طور کامل در block formatting context برای فاصله گذاری عمودی شرکت نمیکنند. اگر نیاز دارید مارجین های عمودی مانند عناصر block رفتار کنند، از `display: flex` (block-level) استفاده کنید.

**عنوان**: چگونه میتوان از ویژگی gap در Flexbox و Grid استفاده کرد و تفاوت های مرورگری در fallback row-gap/column-gap چیست؟  
**سطح دشواری**: سخت  
**دسته‌بندی**: چیدمان

### پاسخ 📄

`gap` (یا `row-gap`/`column-gap`) فاصله بین آیتم ها در flex/grid اضافه میکند (نیازی به margin نیست).

```css
.container {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  gap: 20px;
}
```

نکات مرورگری:

- Safari/IE قدیمی تر: فقط alias `grid-gap` کار میکرد
- پشتیبانی gap در Flexbox: Chrome 84+، Safari 14.1+، Firefox 63+
- در flex، gap به صورت مستقل روی row/column کار میکند

Gap از مشکلات double-margin جلوگیری میکند و چیدمان ها را ساده تر میکند.

### تفاوت gap در Flex و Grid

**در Flexbox:**

- `gap` فاصله بین آیتم ها را اضافه میکند اما به `box-sizing` احترام نمیگذارد
- اگر کانتینر `border-box` باشد، gap میتواند باعث overflow شود
- محاسبه: `width = content + gap` (gap خارج از box-sizing است)

**در Grid:**

- `gap` بخشی از محاسبات grid track است
- به `border-box` احترام میگذارد و overflow ایجاد نمیکند
- محاسبه: gap در فضای موجود grid لحاظ میشود

**راه حل برای Flexbox:**

```css
.flex-container {
  display: flex;
  gap: 20px;
  /* برای جلوگیری از overflow */
  box-sizing: border-box;
  padding: 0; /* یا gap را در محاسبات width لحاظ کنید */
}
```
