## 🧠 سوال 1

**شناسه**: vue-001  
**عنوان**: Vue.js چیست و چه مشکلی را حل می‌کند؟  
**سطح دشواری**: آسان  
**دسته‌بندی**: مبانی Vue

### پاسخ 📄

Vue.js یک فریمورک پیشرونده جاوااسکریپت است که برای ساخت رابط‌های کاربری و اپلیکیشن‌های تک‌صفحه‌ای استفاده می‌شود.

این فریمورک روی **لایه نمایش** تمرکز دارد و به توسعه‌دهندگان اجازه می‌دهد رابط‌های تعاملی را با استفاده از معماری مبتنی بر کامپوننت بسازند.

Vue چندین مشکل رایج فرانت‌اند را حل می‌کند:

- مدیریت state پیچیده UI
- به‌روزرسانی کارآمد DOM
- سازماندهی منطق UI در کامپوننت‌های قابل استفاده مجدد
- ساده‌سازی به‌روزرسانی‌های واکنش‌پذیر داده

Vue از یک **سیستم data-binding واکنش‌پذیر** و **DOM مجازی** برای به‌روزرسانی کارآمد رابط کاربری هنگام تغییر state اپلیکیشن استفاده می‌کند.

از آنجایی که پیشرونده است، Vue می‌تواند به تدریج استفاده شود، از ویجت‌های تعاملی کوچک تا اپلیکیشن‌های تک‌صفحه‌ای کامل.

## 🧠 سوال 2

**شناسه**: vue-002  
**عنوان**: ویژگی‌های اصلی Vue.js چیست؟  
**سطح دشواری**: آسان  
**دسته‌بندی**: مبانی Vue

### پاسخ 📄

Vue.js چندین ویژگی اصلی را ارائه می‌دهد که آن را برای ساخت اپلیکیشن‌های وب مدرن مناسب می‌کند.

ویژگی‌های کلیدی شامل:

- **data binding واکنش‌پذیر**
- **معماری مبتنی بر کامپوننت**
- **رندرینگ DOM مجازی**
- **ویژگی‌های محاسباتی (Computed properties)**
- **نظاره‌گرها (Watchers)**
- **دستوراتی مانند v-if، v-for و v-bind**
- **Vue Router برای مسیریابی**
- **مدیریت state از طریق Pinia یا Vuex**

این ویژگی‌ها به توسعه‌دهندگان کمک می‌کنند اپلیکیشن‌های فرانت‌اند مقیاس‌پذیر و قابل نگهداری بسازند.

## 🧠 سوال 3

**شناسه**: vue-003  
**عنوان**: Vue با React و Angular چه تفاوتی دارد؟  
**سطح دشواری**: آسان  
**دسته‌بندی**: مبانی Vue

### پاسخ 📄

Vue، React و Angular ابزارهای محبوب برای ساخت اپلیکیشن‌های وب مدرن هستند، اما در فلسفه طراحی و پیچیدگی متفاوت‌اند.

Vue روی سادگی و پذیرش تدریجی تمرکز دارد. این فریمورک یک API قابل دسترس و راه‌حل‌های داخلی برای وظایف رایج ارائه می‌دهد.

React در درجه اول یک کتابخانه UI است که روی ساخت رابط‌های مبتنی بر کامپوننت تمرکز دارد. به شدت به اکوسیستم اطراف خود برای مسیریابی و مدیریت state متکی است.

Angular یک فریمورک کامل با بسیاری از ویژگی‌های داخلی مانند dependency injection، مدیریت فرم و یکپارچگی قوی TypeScript است.

به طور کلی:

- Vue بر سادگی و تجربه توسعه‌دهنده تأکید دارد
- React بر انعطاف‌پذیری تأکید دارد
- Angular بر ساختار و ابزارهای سطح سازمانی تأکید دارد

## 🧠 سوال 4

**شناسه**: vue-004  
**عنوان**: Vue instance چیست؟  
**سطح دشواری**: آسان  
**دسته‌بندی**: مبانی Vue

### پاسخ 📄

یک Vue instance ریشه یک اپلیکیشن Vue را نمایش می‌دهد.

در Vue 3، اپلیکیشن‌ها با استفاده از تابع `createApp` ایجاد می‌شوند. Vue instance درخت کامپوننت، state واکنش‌پذیر و فرآیند رندرینگ را مدیریت می‌کند.

این نقطه ورودی است که منطق اپلیکیشن را با DOM متصل می‌کند.

یک Vue instance معمولاً:

- کامپوننت ریشه را نگه می‌دارد
- اپلیکیشن را به یک المان DOM متصل می‌کند
- lifecycle hooks را مدیریت می‌کند
- سیستم واکنش‌پذیری را مقداردهی اولیه می‌کند

مثال:

```js
import { ref, reactive } from 'vue';

const count = ref(0);

const user = reactive({
  name: 'Rasool',
  age: 27,
});

count.value++;
user.age++;
```

## 🧠 سوال 6

**شناسه**: vue-006  
**عنوان**: واکنش‌پذیری Vue چیست و چرا مهم است؟  
**سطح دشواری**: آسان  
**دسته‌بندی**: سیستم واکنش‌پذیری

### پاسخ 📄

سیستم واکنش‌پذیری Vue به طور خودکار وابستگی‌ها بین داده و UI را ردیابی می‌کند.

وقتی داده‌های واکنش‌پذیر تغییر می‌کنند، Vue به طور خودکار بخش‌هایی از DOM را که به آن داده وابسته هستند به‌روزرسانی می‌کند.

این نیاز به دستکاری دستی DOM را از بین می‌برد.

واکنش‌پذیری به توسعه‌دهندگان اجازه می‌دهد روی منطق اپلیکیشن تمرکز کنند در حالی که Vue به طور کارآمد به‌روزرسانی‌های UI را مدیریت می‌کند.

در Vue 3، این سیستم با استفاده از اشیاء Proxy جاوااسکریپت پیاده‌سازی شده است.

## 🧠 سوال 7

**شناسه**: vue-007  
**عنوان**: Vue چگونه وابستگی‌های واکنش‌پذیر را ردیابی می‌کند؟  
**سطح دشواری**: متوسط  
**دسته‌بندی**: سیستم واکنش‌پذیری

### پاسخ 📄

Vue وابستگی‌های واکنش‌پذیر را با استفاده از یک سیستم ردیابی وابستگی ردیابی می‌کند.

وقتی داده‌های واکنش‌پذیر در طول رندرینگ دسترسی پیدا می‌کنند، Vue آن وابستگی را ثبت می‌کند. اگر داده بعداً تغییر کند، Vue می‌داند کدام کامپوننت‌ها نیاز به به‌روزرسانی دارند.

در Vue 3، این سیستم با استفاده از موارد زیر کار می‌کند:

- **track()** برای ثبت وابستگی‌ها
- **trigger()** برای اطلاع‌رسانی به‌روزرسانی‌ها

این مکانیزم تضمین می‌کند که فقط بخش‌های لازم UI دوباره رندر شوند.

## 🧠 سوال 8

**شناسه**: vue-008  
**عنوان**: `ref` و `reactive` در Vue 3 چیست؟  
**سطح دشواری**: آسان  
**دسته‌بندی**: سیستم واکنش‌پذیری

### پاسخ 📄

`ref` و `reactive` APIهایی هستند که برای ایجاد state واکنش‌پذیر در Vue 3 استفاده می‌شوند.

`ref` برای مقادیر ابتدایی یا مراجع واکنش‌پذیر تکی استفاده می‌شود.

`reactive` برای اشیاء واکنش‌پذیر استفاده می‌شود.

هر دو API به Vue اجازه می‌دهند تغییرات را ردیابی کند و UI را به طور خودکار به‌روزرسانی کند.

آن‌ها معمولاً در داخل Composition API استفاده می‌شوند.

مثال:

```js
import { ref, reactive } from 'vue';

const count = ref(0);

const user = reactive({
  name: 'Rasool',
  age: 27,
});

count.value++;
user.age++;
```

## 🧠 سوال 9

**شناسه**: vue-009  
**عنوان**: تفاوت `ref` و `reactive` چیست؟  
**سطح دشواری**: متوسط  
**دسته‌بندی**: سیستم واکنش‌پذیری

### پاسخ 📄

تفاوت اصلی بین `ref` و `reactive` در نحوه ذخیره و دسترسی به داده‌های واکنش‌پذیر است.

`ref` یک مقدار را در داخل یک شیء با ویژگی `.value` می‌پیچد.

`reactive` مستقیماً یک شیء را به یک proxy واکنش‌پذیر تبدیل می‌کند.

به طور کلی:

- `ref` برای مقادیر ابتدایی بهتر کار می‌کند
- `reactive` برای اشیاء بهتر کار می‌کند

هر دو با سیستم ردیابی وابستگی Vue یکپارچه می‌شوند.

## 🧠 سوال 10

**شناسه**: vue-010  
**عنوان**: ویژگی‌های محاسباتی (Computed properties) چیست و با متدها چه تفاوتی دارند؟  
**سطح دشواری**: آسان  
**دسته‌بندی**: سیستم واکنش‌پذیری

### پاسخ 📄

ویژگی‌های محاسباتی مقادیر واکنش‌پذیری هستند که از داده‌های واکنش‌پذیر دیگر مشتق می‌شوند.

آن‌ها به طور خودکار وقتی وابستگی‌هایشان تغییر می‌کنند به‌روزرسانی می‌شوند.

برخلاف متدها، ویژگی‌های محاسباتی بر اساس وابستگی‌هایشان **کش** می‌شوند. این بدان معناست که فقط در صورت لزوم دوباره ارزیابی می‌شوند.

متدها هر بار که فراخوانی می‌شوند اجرا می‌شوند، در حالی که ویژگی‌های محاسباتی فقط زمانی دوباره محاسبه می‌شوند که وابستگی‌هایشان تغییر کنند.

مثال:

```js
import { computed, ref } from 'vue';

const price = ref(100);

const tax = computed(() => {
  return price.value * 0.2;
});
```
