## 🧠 سوال 1

**شناسه**: vue-001  
**عنوان**: Vue.js چیست و چه مشکلی را حل می‌کند؟  
**سطح دشواری**: آسان  
**دسته‌بندی**: مبانی Vue

### پاسخ 📄

Vue.js یک فریمورک پیشرونده جاوااسکریپت است که برای ساخت رابط‌های کاربری و اپلیکیشن‌های تک‌صفحه‌ای استفاده می‌شود.

این فریمورک روی **لایه نمایش** تمرکز دارد و به توسعه‌دهندگان اجازه می‌دهد رابط‌های تعاملی را با استفاده از معماری مبتنی بر کامپوننت بسازند.

Vue چندین مشکل رایج فرانت‌اند را حل می‌کند:

- مدیریت state پیچیده UI
- به‌روزرسانی کارآمد DOM
- سازماندهی منطق UI در کامپوننت‌های قابل استفاده مجدد
- ساده‌سازی به‌روزرسانی‌های واکنش‌پذیر داده

Vue از یک **سیستم data-binding واکنش‌پذیر** و **DOM مجازی** برای به‌روزرسانی کارآمد رابط کاربری هنگام تغییر state اپلیکیشن استفاده می‌کند.

از آنجایی که پیشرونده است، Vue می‌تواند به تدریج استفاده شود، از ویجت‌های تعاملی کوچک تا اپلیکیشن‌های تک‌صفحه‌ای کامل.

### پاسخ 📄

Vue، React و Angular ابزارهای محبوب برای ساخت اپلیکیشن‌های وب مدرن هستند، اما در فلسفه طراحی و پیچیدگی متفاوت‌اند.

Angular یک فریمورک کامل با بسیاری از ویژگی‌های داخلی مانند dependency injection، مدیریت فرم و یکپارچگی قوی TypeScript است.

به طور کلی:

- Vue بر سادگی و تجربه توسعه‌دهنده تأکید دارد
- React بر انعطاف‌پذیری تأکید دارد

## 🧠 سوال 4

**شناسه**: vue-004  
**عنوان**: Vue instance چیست؟  
**سطح دشواری**: آسان  
**دسته‌بندی**: مبانی Vue

### پاسخ 📄

یک Vue instance ریشه یک اپلیکیشن Vue را نمایش می‌دهد.

در Vue 3، اپلیکیشن‌ها با استفاده از تابع `createApp` ایجاد می‌شوند. Vue instance درخت کامپوننت، state واکنش‌پذیر و فرآیند رندرینگ را مدیریت می‌کند.

این نقطه ورودی است که منطق اپلیکیشن را با DOM متصل می‌کند.

یک Vue instance معمولاً:

- کامپوننت ریشه را نگه می‌دارد
- اپلیکیشن را به یک المان DOM متصل می‌کند
- lifecycle hooks را مدیریت می‌کند
- سیستم واکنش‌پذیری را مقداردهی اولیه می‌کند

مثال:

```js
import { createApp } from 'vue';
import App from './App.vue';

const app = createApp(App);

app.use(router);
app.use(pinia);

app.mount('#app');
```

## 🧠 سوال 8

**شناسه**: vue-008  
**عنوان**: `ref` و `reactive` در Vue 3 چیست؟  
**سطح دشواری**: آسان  
**دسته‌بندی**: سیستم واکنش‌پذیری

### پاسخ 📄

`ref` و `reactive` APIهایی هستند که برای ایجاد state واکنش‌پذیر در Vue 3 استفاده می‌شوند.

`ref` برای مقادیر ابتدایی یا مراجع واکنش‌پذیر تکی استفاده می‌شود.

`reactive` برای اشیاء واکنش‌پذیر استفاده می‌شود.

هر دو API به Vue اجازه می‌دهند تغییرات را ردیابی کند و UI را به طور خودکار به‌روزرسانی کند.

آن‌ها معمولاً در داخل Composition API استفاده می‌شوند.

مثال:

```js
import { ref, reactive } from 'vue';

const count = ref(0);

const user = reactive({
  name: 'Rasool',
  age: 27,
});

count.value++;
user.age++;
```

## 🧠 سوال 9

**شناسه**: vue-009  
**عنوان**: تفاوت `ref` و `reactive` چیست؟  
**سطح دشواری**: متوسط  
**دسته‌بندی**: سیستم واکنش‌پذیری

### پاسخ 📄

تفاوت اصلی بین `ref` و `reactive` در نحوه ذخیره و دسترسی به داده‌های واکنش‌پذیر است.

`ref` یک مقدار را در داخل یک شیء با ویژگی `.value` می‌پیچد.

`reactive` مستقیماً یک شیء را به یک proxy واکنش‌پذیر تبدیل می‌کند.

به طور کلی:

- `ref` برای مقادیر ابتدایی بهتر کار می‌کند
- `reactive` برای اشیاء بهتر کار می‌کند

هر دو با سیستم ردیابی وابستگی Vue یکپارچه می‌شوند.

مثال:

```js
import { ref, reactive } from 'vue';

// ref — دسترسی از طریق .value
const count = ref(0);
count.value++;

// reactive — نیازی به .value نیست
const user = reactive({ name: 'Rasool', age: 27 });
user.age++;

// هشدار: destructuring از reactive واکنش‌پذیری را از بین می‌برد
const { age } = user; // دیگر واکنش‌پذیر نیست
```

```js

});
```

## 🧠 سوال 17

**شناسه**: vue-017  
**عنوان**: تابع `setup()` در Vue چیست؟  
**سطح دشواری**: متوسط  
**دسته‌بندی**: Composition API

### پاسخ 📄

تابع `setup()` نقطه ورودی برای استفاده از Composition API در داخل یک کامپوننت Vue است.

این تابع قبل از ایجاد کامپوننت اجرا میشود و مسئول مقداردهی اولیه state واکنشپذیر، مقادیر محاسباتی و توابع مورد استفاده توسط کامپوننت است.

هر چیزی که از `setup()` برگردانده شود در داخل قالب کامپوننت در دسترس قرار میگیرد.

تابع `setup()` دو آرگومان دریافت میکند:

- `props`
- `context`

این تابع معمولاً برای تعریف منطق کامپوننت در اپلیکیشنهای Vue 3 استفاده میشود.

مثال (سوال 16-17):

```js
//Composition API با setup()

import { ref } from 'vue';

export default {
  setup() {
    const count = ref(0);

    const increment = () => {
      count.value++;
    };

    return { count, increment };
```

```js
// toRefs — بدون از دست دادن واکنش‌پذیری destructure کنید
const { name, age: ageRef } = toRefs(state);
```

### پاسخ 📄

معمولاً برای موارد زیر استفاده می‌شوند:

- بررسی احراز هویت قبل از دسترسی به route‌های محافظت شده
- هدایت کاربران احراز هویت نشده به صفحه ورود
- جلوگیری از ناوبری هنگامی که تغییرات ذخیره نشده وجود دارد

Vue Router سه نوع guard ارائه می‌دهد:

- **Guard‌های سراسری** — `router.beforeEach`، `router.afterEach`
- **Guard‌های per-route** — تعریف شده در پیکربندی route
- **Guard‌های درون کامپوننت** — `onBeforeRouteLeave`، `onBeforeRouteUpdate`

مثال:

```js
// Navigation guard سراسری
router.beforeEach((to, from) => {
  const isAuthenticated = !!localStorage.getItem('token');

  if (to.meta.requiresAuth && !isAuthenticated) {
    return { path: '/login' };
  }
});
```

این hook به شما اجازه می‌دهد error boundary‌های سطح کامپوننت پیاده‌سازی کنید و UI جایگزین نمایش دهید.

```js
import { onErrorCaptured, ref } from 'vue';

const hasError = ref(false);
const errorMessage = ref('');

  errorMessage.value = err.message;
  return false; // از انتشار بیشتر خطا جلوگیری می‌کند
});
```

```js
// Handler خطای سراسری
app.config.errorHandler = (err, instance, info) => {

};
```

## 🧠 سوال 48

**شناسه**: vue-048
**عنوان**: `shallowRef()` و `shallowReactive()` در Vue چیست؟
**سطح دشواری**: سخت
**دسته‌بندی**: سیستم واکنش‌پذیری

### پاسخ 📄

`shallowRef()` و `shallowReactive()` جایگزین‌های performance-focused برای `ref()` و `reactive()` هستند.

`shallowRef()` فقط `.value` سطح بالا را واکنش‌پذیر می‌کند. اشیاء تو در تو ردیابی نمی‌شوند.

`shallowReactive()` فقط property‌های مستقیم یک شیء را واکنش‌پذیر می‌کند. property‌های تو در تو ردیابی نمی‌شوند.

این‌ها هنگام کار با ساختارهای داده بزرگ یا کتابخانه‌های خارجی مفید هستند که واکنش‌پذیری عمیق باعث سربار غیرضروری می‌شود.

مثال:

```js
import { shallowRef, shallowReactive } from 'vue';

const state = shallowRef({ nested: { count: 0 } });

// واکنش‌پذیری را فعال می‌کند (مقدار سطح بالا را جایگزین می‌کند)
state.value = { nested: { count: 1 } };

// واکنش‌پذیری را فعال نمی‌کند (property تو در تو را mutate می‌کند)
state.value.nested.count = 2;

// shallowReactive
const form = shallowReactive({ user: { name: 'Rasool' } });
form.user = { name: 'Ali' }; // واکنش‌پذیری را فعال می‌کند
form.user.name = 'Ali'; // واکنش‌پذیری را فعال نمی‌کند
```

**عنوان**: `defineExpose()` در Vue 3 چیست و چرا به آن نیاز است؟
**سطح دشواری**: متوسط
**دسته‌بندی**: Composition API

### پاسخ 📄

به طور پیش‌فرض، کامپوننت‌هایی که از `<script setup>` استفاده می‌کنند هیچ state داخلی یا متدی را به کامپوننت‌های والد خود نمایش نمی‌دهند.

`defineExpose()` به صراحت تعریف می‌کند که یک کامپوننت فرزند چه چیزی را از طریق template ref‌ها به کامپوننت‌های والد نمایش می‌دهد.

این اصل encapsulation را دنبال می‌کند — فقط بخش‌های مورد نظر API کامپوننت از خارج قابل دسترسی هستند.

مثال:

```vue
<!-- Child.vue -->
<script setup>
import { ref } from 'vue';

const count = ref(0);

function reset() {
  count.value = 0;
}

defineExpose({ count, reset });
</script>
```

```vue
<!-- Parent.vue -->
<script setup>
import { ref } from 'vue';
import Child from './Child.vue';

const childRef = ref(null);

function resetChild() {
  childRef.value.reset();
}
</script>

<template>
  <Child ref="childRef" />
  <button @click="resetChild">بازنشانی</button>
</template>
```

## 🧠 سوال 50

**شناسه**: vue-050
**عنوان**: ناوبری برنامه‌نویسی (Programmatic navigation) در Vue Router چیست؟
**سطح دشواری**: متوسط
**دسته‌بندی**: مسیریابی (Routing)

### پاسخ 📄

ناوبری برنامه‌نویسی به شما اجازه می‌دهد route‌ها را با استفاده از JavaScript به جای `<RouterLink>` تغییر دهید.

این برای redirect کردن بعد از ارسال فرم، ورود، یا هر منطق ناوبری شرطی مفید است.

Vue Router دو متد اصلی ارائه می‌دهد:

- `router.push()` — به یک route جدید ناوبری می‌کند و یک entry به history مرورگر اضافه می‌کند
- `router.replace()` — به یک route جدید ناوبری می‌کند اما entry فعلی history را جایگزین می‌کند

می‌توانید یک path string یا یک route location object با name، params یا query ارسال کنید.

مثال:

```js
import { useRouter } from 'vue-router';

const router = useRouter();

// ناوبری بر اساس path
router.push('/dashboard');

// ناوبری با route نام‌دار و params
router.push({ name: 'Profile', params: { id: 42 } });

// ناوبری با query string
router.push({ path: '/search', query: { q: 'vue' } });

// جایگزین کردن route فعلی (بدون entry history جدید)
router.replace('/login');
```
