## 🧠 سوال 1

**شناسه**: vue-001  
**عنوان**: Vue.js چیست و چه مشکلی را حل می‌کند؟  
**سطح دشواری**: آسان  
**دسته‌بندی**: مبانی Vue

### پاسخ 📄

Vue.js یک فریمورک پیشرونده جاوااسکریپت است که برای ساخت رابط‌های کاربری و اپلیکیشن‌های تک‌صفحه‌ای استفاده می‌شود.

این فریمورک روی **لایه نمایش** تمرکز دارد و به توسعه‌دهندگان اجازه می‌دهد رابط‌های تعاملی را با استفاده از معماری مبتنی بر کامپوننت بسازند.

Vue چندین مشکل رایج فرانت‌اند را حل می‌کند:

- مدیریت state پیچیده UI
- به‌روزرسانی کارآمد DOM
- سازماندهی منطق UI در کامپوننت‌های قابل استفاده مجدد
- ساده‌سازی به‌روزرسانی‌های واکنش‌پذیر داده

Vue از یک **سیستم data-binding واکنش‌پذیر** و **DOM مجازی** برای به‌روزرسانی کارآمد رابط کاربری هنگام تغییر state اپلیکیشن استفاده می‌کند.

از آنجایی که پیشرونده است، Vue می‌تواند به تدریج استفاده شود، از ویجت‌های تعاملی کوچک تا اپلیکیشن‌های تک‌صفحه‌ای کامل.

## 🧠 سوال 2

**شناسه**: vue-002  
**عنوان**: ویژگی‌های اصلی Vue.js چیست؟  
**سطح دشواری**: آسان  
**دسته‌بندی**: مبانی Vue

### پاسخ 📄

Vue.js چندین ویژگی اصلی را ارائه می‌دهد که آن را برای ساخت اپلیکیشن‌های وب مدرن مناسب می‌کند.

ویژگی‌های کلیدی شامل:

- **data binding واکنش‌پذیر**
- **معماری مبتنی بر کامپوننت**
- **رندرینگ DOM مجازی**
- **ویژگی‌های محاسباتی (Computed properties)**
- **نظاره‌گرها (Watchers)**
- **دستوراتی مانند v-if، v-for و v-bind**
- **Vue Router برای مسیریابی**
- **مدیریت state از طریق Pinia یا Vuex**

این ویژگی‌ها به توسعه‌دهندگان کمک می‌کنند اپلیکیشن‌های فرانت‌اند مقیاس‌پذیر و قابل نگهداری بسازند.

## 🧠 سوال 3

**شناسه**: vue-003  
**عنوان**: Vue با React و Angular چه تفاوتی دارد؟  
**سطح دشواری**: آسان  
**دسته‌بندی**: مبانی Vue

### پاسخ 📄

Vue، React و Angular ابزارهای محبوب برای ساخت اپلیکیشن‌های وب مدرن هستند، اما در فلسفه طراحی و پیچیدگی متفاوت‌اند.

Vue روی سادگی و پذیرش تدریجی تمرکز دارد. این فریمورک یک API قابل دسترس و راه‌حل‌های داخلی برای وظایف رایج ارائه می‌دهد.

React در درجه اول یک کتابخانه UI است که روی ساخت رابط‌های مبتنی بر کامپوننت تمرکز دارد. به شدت به اکوسیستم اطراف خود برای مسیریابی و مدیریت state متکی است.

Angular یک فریمورک کامل با بسیاری از ویژگی‌های داخلی مانند dependency injection، مدیریت فرم و یکپارچگی قوی TypeScript است.

به طور کلی:

- Vue بر سادگی و تجربه توسعه‌دهنده تأکید دارد
- React بر انعطاف‌پذیری تأکید دارد
- Angular بر ساختار و ابزارهای سطح سازمانی تأکید دارد

## 🧠 سوال 4

**شناسه**: vue-004  
**عنوان**: Vue instance چیست؟  
**سطح دشواری**: آسان  
**دسته‌بندی**: مبانی Vue

### پاسخ 📄

یک Vue instance ریشه یک اپلیکیشن Vue را نمایش می‌دهد.

در Vue 3، اپلیکیشن‌ها با استفاده از تابع `createApp` ایجاد می‌شوند. Vue instance درخت کامپوننت، state واکنش‌پذیر و فرآیند رندرینگ را مدیریت می‌کند.

این نقطه ورودی است که منطق اپلیکیشن را با DOM متصل می‌کند.

یک Vue instance معمولاً:

- کامپوننت ریشه را نگه می‌دارد
- اپلیکیشن را به یک المان DOM متصل می‌کند
- lifecycle hooks را مدیریت می‌کند
- سیستم واکنش‌پذیری را مقداردهی اولیه می‌کند

مثال:

```js
import { ref, reactive } from 'vue';

const count = ref(0);

const user = reactive({
  name: 'Rasool',
  age: 27,
});

count.value++;
user.age++;
```

## 🧠 سوال 5

**شناسه**: vue-005  
**عنوان**: تفاوت Vue 2 و Vue 3 چیست؟  
**سطح دشواری**: آسان  
**دسته‌بندی**: مبانی Vue

### پاسخ 📄

Vue 3 چندین بهبود نسبت به Vue 2 معرفی کرد.

تفاوت‌های اصلی شامل:

- **Composition API** برای استفاده مجدد بهتر منطق
- **سیستم واکنش‌پذیری جدید مبتنی بر Proxy**
- **پشتیبانی بهبود یافته از TypeScript**
- **عملکرد بهتر و اندازه bundle کوچکتر**
- **پشتیبانی از Fragment برای چندین المان ریشه**
- **کامپوننت‌های Teleport و Suspense**

Vue 3 معماری انعطاف‌پذیرتر و مقیاس‌پذیرتری نسبت به Vue 2 ارائه می‌دهد.
